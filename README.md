# HappyBirthday

友人の誕生日に見せる、スマートフォン向けの静的なネタサイトです。架空の「国際生誕記念機構」による**生誕記念特別中継**という体裁で、国家式典のように過剰に本格的な公式サイトに見えますが、祝辞者情報を確認すると、本人が誰も知らないという構成になっています。

HTML・CSS・JavaScriptだけで動作し、外部ライブラリ、外部フォント、外部画像、外部API、ビルド作業は不要です。動画がまだなくてもプレースホルダーで最後まで画面遷移を確認できます。

## 起動方法

1. このリポジトリをダウンロードまたはクローンします。
2. `index.html` をダブルクリックしてブラウザで開きます。
3. 「中継を開始する」を押します。

ローカルサーバーは不要です。公開前にブラウザの開発者ツールでスマートフォン表示も確認することをおすすめします。

## 動画ファイルの入れ方

リポジトリ直下に `videos` フォルダを作り、次の名前でMP4動画を配置してください。

```text
HappyBirthday/
├─ index.html
├─ style.css
├─ script.js
└─ videos/
   ├─ 01_sakura.mp4
   ├─ 02_rice_farmer.mp4
   ├─ 03_rival.mp4
   ├─ 04_coerced.mp4
   ├─ 05_future.mp4
   ├─ 06_apology.mp4
   ├─ 07_business.mp4
   ├─ 08_cult.mp4
   ├─ 09_wrong_name.mp4
   ├─ 10_knows_too_much.mp4
   └─ 11_final.mp4
```

通常メッセージは `01`〜`10`、最後に知人から届く体裁の動画は `11_final.mp4` です。ファイルがない動画は自動的にプレースホルダーへ切り替わります。

## 友人の名前を変更する

`script.js` 冒頭の次の1行だけを変更してください。

```javascript
const birthdayPerson = "ケイスケ";
```

たとえば `const birthdayPerson = "太郎";` とすると、サイト内の対象箇所（開幕画面の「太郎 氏」、最終画面の「太郎さんへ」）がすべて置き換わります。HTMLへ名前を直接書く必要はありません。名前は `data-birthday-person` の付いた要素へ自動的に流し込まれます。

## 動画の順番や送信者情報を変更する

`script.js` の `messages` 配列が、通常動画10件の再生順と表示内容を管理しています。順番を変える場合は、配列内のオブジェクトを情報ごと移動してください。

```javascript
{
  video: "videos/01_sakura.mp4",
  title: "MESSAGE 01",
  sender: "さくらさん",
  relation: "本人との関係は確認できませんでした",
  status: "お祝いの意志：確認済み"
}
```

`video` だけを入れ替えると表示情報と映像がずれるため、オブジェクト単位で変更してください。お祝い人数の進み方は、その下にある `peopleCounts` 配列で変更できます。

## デザインと演出の段階

中継が進むほど、少しずつ不穏になっていく演出を入れています。`script.js` の `getPhase()` が現在の段階を決め、`#app` の `data-phase` 属性へ書き込みます。色や走査線の濃さは `style.css` 側の `--signal` と `--tension` が受け取ります。

| 段階 | 対象 | 見た目 |
| --- | --- | --- |
| `calm` | 開幕・接続・MESSAGE 01〜04 | 状態色は青（steel）。走査線なし |
| `caution` | MESSAGE 05〜07 | 状態色が琥珀色へ。走査線がうっすら出る |
| `alert` | MESSAGE 08〜10・結果・検索 | 状態色が赤へ。走査線と周辺減光が最も濃い |

最終画面で `#birthday-ending` が表示されると、CSSの `:has()` が反応して暖色の明るい配色へ反転し、紙吹雪が降ります。JavaScript側の追加処理は不要です。

段階の境目を変えたい場合は `getPhase()` のしきい値（`4` と `7`）と、`getTransitionLines()` のしきい値を揃えて変更してください。

## 公開方法

このサイトは静的ファイルだけなので、一般的な静的ホスティングへそのまま公開できます。

### GitHub Pages

1. GitHubへリポジトリをプッシュします。
2. リポジトリの **Settings → Pages** を開きます。
3. **Build and deployment** で **Deploy from a branch** を選びます。
4. 公開するブランチ（通常は `main`）と `/ (root)` を選んで保存します。

数分後、Pages欄に公開URLが表示されます。

### Netlify

Netlifyへログインし、リポジトリを連携して新しいサイトを作成します。Build commandは空欄、Publish directoryはリポジトリ直下を指定します。フォルダを画面へドラッグ＆ドロップして公開する方法も利用できます。

### Vercel

Vercelへログインしてリポジトリをインポートします。Framework Presetは **Other**、Build Commandは空欄、Output Directoryは `.`（リポジトリ直下）としてデプロイします。

## 動画が再生できない場合

次を順番に確認してください。

- `videos` フォルダと動画ファイル名が上記と完全に一致しているか
- 大文字・小文字、アンダースコア、拡張子が間違っていないか
- MP4ファイルが破損しておらず、ほかの動画プレーヤーで開けるか
- ブラウザが動画の映像・音声コーデックに対応しているか
- 公開先へ動画ファイルも一緒にアップロードされているか
- 動画ファイルが大きすぎないか、通信が途中で切れていないか
- ブラウザの自動再生制限が働いた場合、動画内の再生ボタンを押したか

互換性を高めるため、MP4の映像は **H.264形式** を推奨します。音声はAAC形式が広く対応しています。スマートフォン向けには、画質と通信量のバランスを見ながら動画を圧縮してください。

## 公開時の注意

公開URLは第三者も閲覧できる可能性があります。本名、顔、住所、学校名、勤務先、行動履歴などの個人情報や、本人が不快に感じる内容を載せすぎないでください。出演者にも公開範囲を説明し、必要な許可を得てから公開してください。

限定公開にしたい場合は、URLの共有範囲を絞るか、アクセス制限を設定できるホスティングサービスを利用してください。
