# HappyBirthday

友人の誕生日に見せる、スマートフォン向けの静的なネタサイトです。「ケイスケ生誕祭 2026」という期間限定キャンペーンサイトの体裁で、明るく上品にお祝いメッセージ動画を順番に届けます。出演者が本人の知らない人ばかりであることは、サイト側では説明せず、動画から自然に伝わる構成です。

HTML・CSS・JavaScriptだけで動作し、外部ライブラリ、外部フォント、外部画像、外部API、ビルド作業は不要です。動画がまだ無い枠は、自動的に「映像を準備中です」の表示になり、最後まで画面遷移を確認できます。

## ファイル構成

```text
HappyBirthday/
├─ index.html      … 画面と文言
├─ style.css       … 明るいお祝いテーマ（1ファイルに統合）
├─ script.js       … 画面遷移・動画・BGM（1ファイルに統合）
├─ Keisuke.mp3     … 入場BGM
└─ *.mp4           … お祝いメッセージ動画（リポジトリ直下）
```

## 起動方法

1. このリポジトリをダウンロードまたはクローンします。
2. `index.html` をダブルクリックしてブラウザで開きます。
3. 「お祝いメッセージを見る」を押します。

ローカルサーバーは不要です。公開前にブラウザの開発者ツールでスマートフォン表示も確認することをおすすめします。

## BGMについて

開始ボタンを押すと `Keisuke.mp3` が再生され、最初の動画が始まると自動でフェードアウトします。ブラウザの自動再生制限により、ボタン操作前には再生されません。曲を差し替える場合は、同名のMP3ファイルを置き換えてください。

## 動画と再生順

`script.js` の `messages` 配列が、全10通の再生順とメッセージカードの内容を管理しています。現在の割り当ては次のとおりです。

| 順番 | 動画ファイル | 状態 |
| --- | --- | --- |
| 01 | `さくらさん_8秒まで_0.9倍速.mp4` | あり |
| 02 | `コメ農家の野口さん.mp4` | あり |
| 03 | `ライバル.mp4` | あり |
| 04 | `脅迫.mp4` | あり |
| 05 | `未来から.mp4` | あり |
| 06 | `893.mp4` | あり |
| 07〜10 | `videos/07_business.mp4` ほか | 未配置（準備中表示） |
| 最後 | `videos/11_final.mp4` | 未配置（代替メッセージ表示） |

動画を追加するときは、ファイルを置いてから `messages` 配列の `video` をそのファイル名に変更してください。順番を入れ替えるときは、オブジェクト単位で移動すると表示内容と映像がずれません。

各メッセージのカードは `sender`（お名前）・`theme`（テーマ）・`comment`（ひとこと）の3項目です。ネタばらしになる表現（知人ではない・照合できない等）は書かず、普通のゲスト紹介として書くのがこのサイトの流儀です。

最後の動画 `videos/11_final.mp4` を置くと、検索演出のあとに再生され、終了後にエンディングへ進みます。無い場合は、`index.html` の `#final-fallback` にある文面が表示されます。

## 友人の名前を変更する

`script.js` 冒頭の次の1行だけを変更してください。

```javascript
const birthdayPerson = "ケイスケ";
```

サイト内の対象箇所（開幕・エンディングの宛名など）がすべて置き換わります。名前は `data-birthday-person` の付いた要素へ自動的に流し込まれます。なお「ケイスケ生誕祭 実行委員会」などの飾り文言は `index.html` に直接書かれているため、別の名前で使う場合はあわせて編集してください。

## お祝い人数の進み方

`script.js` の `peopleCounts` 配列で、各メッセージ完了時点の「お祝い人数」を変更できます。

## 公開方法

このサイトは静的ファイルだけなので、一般的な静的ホスティングへそのまま公開できます。

### GitHub Pages

1. GitHubへリポジトリをプッシュします。
2. リポジトリの **Settings → Pages** を開きます。
3. **Build and deployment** で **Deploy from a branch** を選びます。
4. 公開するブランチ（通常は `main`）と `/ (root)` を選んで保存します。

数分後、Pages欄に公開URLが表示されます。

### Netlify / Vercel

リポジトリを連携し、Build commandは空欄、公開ディレクトリはリポジトリ直下を指定してください。

## 動画が再生できない場合

次を順番に確認してください。

- `messages` 配列の `video` と実際のファイル名が完全に一致しているか（日本語ファイル名も使えます）
- MP4ファイルが破損しておらず、ほかの動画プレーヤーで開けるか
- ブラウザが動画の映像・音声コーデックに対応しているか（映像は **H.264**、音声は **AAC** を推奨）
- 公開先へ動画ファイルも一緒にアップロードされているか
- ブラウザの自動再生制限が働いた場合、動画内の再生ボタンを押したか

## 公開時の注意

公開URLは第三者も閲覧できる可能性があります。本名、顔、住所、学校名、勤務先、行動履歴などの個人情報や、本人が不快に感じる内容を載せすぎないでください。出演者にも公開範囲を説明し、必要な許可を得てから公開してください。

限定公開にしたい場合は、URLの共有範囲を絞るか、アクセス制限を設定できるホスティングサービスを利用してください。
